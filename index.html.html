01 · Root-Finding

Bisection Method

The idea

If f(a) and f(b) have opposite signs and f is continuous, a root lies in [a, b]. Repeatedly halve the interval and keep the half that still brackets the root.

Each iteration:

  1. Midpoint c = (a + b) / 2
  2. Evaluate f(c)
  3. If f(a)·f(c) < 0 → root in [a, c], set b = c. Else → root in [c, b], set a = c.
  4. Repeat until the interval is small enough.

Key facts

  • Always converges from a valid bracket — but slowly (linear).
  • Error after n steps ≤ (b − a) / 2ⁿ.
  • Iterations for tolerance ε:  n ≥ log₂((b − a) / ε).
Exam traps
  • You MUST start with a sign change: f(a)·f(b) < 0.
  • Compute the new midpoint from the updated a and b.

Worked example — root of f(x) = x³ − x − 2 on [1, 2].   f(1) = −2, f(2) = 4 → sign change ✓

02 · Root-Finding

Newton-Raphson Method

The idea

Use the tangent line at the current guess to jump to a better one. Very fast when it works.

xₙ₊₁ = xₙ − f(xₙ) / f′(xₙ)

Key facts

  • Quadratic convergence — roughly doubles correct digits each step near the root.
  • Needs the derivative f′(x).
  • Can fail or diverge if f′(xₙ) ≈ 0 or the start guess is poor.
Exam traps
  • Write f′(x) symbolically before plugging in numbers.
  • Carry enough decimals — precision is the whole point.

Worked example — f(x) = x³ − x − 2, so f′(x) = 3x² − 1. Start x₀ = 1.5.

x₁ = 1.5 − (−0.125)/(5.75) = 1.521739 x₂ = 1.521739 − (0.002214)/(5.947068) = 1.521367

Two steps from a rough guess → 4-decimal accuracy (true root ≈ 1.521380).

03 · Root-Finding

Secant Method

The idea

Like Newton-Raphson, but approximate the derivative using two previous points — so you need no f′(x).

xₙ₊₁ = xₙ − f(xₙ)·(xₙ − xₙ₋₁) / (f(xₙ) − f(xₙ₋₁))

Needs two starting points. Converges almost as fast as Newton (order ≈ 1.618), with no derivative required.

xₙ₊₁ = g(xₙ) converges near a root if |g′(x)| < 1
x³ − 7x + 1 = 0 → rearrange as x = g(x) = (x³ + 1) / 7

04 · Linear Systems · Direct

Gaussian Elimination

The idea

Use row operations to make the augmented matrix upper-triangular (forward elimination), then solve bottom-up (back substitution).

Worked example

2x + y − z = 8 −3x − y + 2z = −11 −2x + y + 2z = −3

Augmented matrix → clear column 1, then column 2:

[ 2 1 −1 | 8 ] [ 0 0.5 0.5 | 1 ] [ 0 0 −1 | 1 ] → −z = 1

Back substitution: −z = 1 → z = −1;   0.5y + 0.5(−1) = 1 → y = 3;   2x + 3 − (−1) = 8 → x = 2.

Solution: (x, y, z) = (2, 3, −1)
Exam traps
  • Partial pivoting: if a pivot is 0 or tiny, swap rows so the largest-magnitude entry becomes the pivot — avoids dividing by ~0 and cuts rounding error.
  • Apply each row operation across the whole row, including the right-hand side.

A·x = b → x = A⁻¹ · b (only if det A ≠ 0)
⎡ 3 −1 2 ⎤ ⎡ −0.5714 0.7143 1 ⎤ ⎢ 1 2 3 ⎥ A⁻¹ = ⎢ −1 1 1 ⎥ det A = −7 ⎣ 2 −2 −1 ⎦ ⎣ 0.8571 −0.5714 −1 ⎦
x = A⁻¹·b = A⁻¹·(12, 11, 2)ᵀ = (3, 1, 2)
05 · Linear Systems · Iterative

Jacobi Iteration

The idea

Solve each equation for its diagonal variable, then update all variables at once using values from the previous iteration only.

Convergence: guaranteed if the matrix is diagonally dominant (each |diagonal| ≥ sum of |other entries in that row|). Always state this.

4x − y + z = 7 x = (7 + y − z) / 4 4x − 8y + z = −21 ⟹ y = (21 + 4x + z) / 8 −2x + y + 5z = 15 z = (15 + 2x − y) / 5

Start (0, 0, 0) — all right-hand sides use old values:

06 · Linear Systems · Iterative

Gauss-Seidel Iteration

The idea

Same as Jacobi, but use new values as soon as they're available within the same iteration. Usually converges faster.

Same system, starting (0, 0, 0):

x = (7 + y − z)/4 ← uses latest y, z y = (21 + 4x + z)/8 ← uses the x just computed z = (15 + 2x − y)/5 ← uses the new x and y

Say this on the exam

Jacobi updates everything at once from the previous step; Gauss-Seidel plugs in each new value immediately, so later variables in the same sweep already benefit from updated earlier ones.

Practice Problems

Try each cold, then reveal the worked solution. Filter by difficulty:

P1BisectionEasy

Do 2 iterations of bisection for f(x) = x² − 2 on [1, 2]. Give the final interval.

Solution

f(1) = −1, f(2) = 2.

c₁ = 1.5 → f = +0.25 → [1, 1.5].   c₂ = 1.25 → f = −0.4375 → [1.25, 1.5] (true √2 ≈ 1.414).

P2BisectionMedium

How many bisection iterations are needed to locate a root in [2, 3] to within tolerance ε = 0.001?

Solution

n ≥ log₂((b − a)/ε) = log₂(1/0.001) = log₂(1000) = 9.97 → n = 10 iterations.

P3BisectionHard

f(x) = x³ − 4x + 1 has a root in [0, 1]. Do 3 iterations; state the bracket.

Solution

f(0) = 1, f(1) = −2.

c₁ = 0.5 → −0.875 → [0, 0.5].   c₂ = 0.25 → +0.0156 → [0.25, 0.5].   c₃ = 0.375 → −0.447 → [0.25, 0.375] (true root ≈ 0.254).

P4Newton-RaphsonEasy

For f(x) = x² − 2, start x₀ = 1.5. Compute x₁.

Solution

f′(x) = 2x.   x₁ = 1.5 − (0.25)/(3) = 1.41667 (true √2 ≈ 1.41421).

P5Newton-RaphsonMedium

P6Newton-RaphsonHard

For f(x) = x·eˣ − 1, start x₀ = 0.5. Compute x₁ and x₂.

Solution

f(0.5) = −0.175639, f′(0.5) = 2.473081 x₁ = 0.5 + 0.071019 = 0.571019 f(0.571019) = 0.010783, f′ = 2.780993 x₂ = 0.571019 − 0.003878 = 0.567141

Last-Minute Checklist

  • Bisection — confirm the sign change; midpoint from current a, b.
  • Newton — write f′(x) first; watch for f′(xₙ) ≈ 0.
  • Gaussian — use partial pivoting if a pivot is small/zero; operate on whole rows.
  • Jacobi vs Gauss-Seidel — state diagonal dominance; Gauss-Seidel reuses fresh values immediately.
  • Precision — carry 4–6 decimals in iterative work unless told otherwise.
